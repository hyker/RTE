Restricted and Trusted Environment (RTE)

This client runs entirely in your browser — all verification and encryption happen locally on your device. You can inspect the source: browser-client.js, tdx-quote-verifier.js, utils.js
The security of this system depends on the integrity of this client code. If you do not fully trust the server delivering these files, you should audit the source yourself or compare it against a known-good copy before proceeding. The full project source is available at github.com/hyker/RTE. The served files browser-client.js, tdx-quote-verifier.js, and utils.js are the un-bundled originals — bundle.js is produced from them using esbuild and can be reproduced from the project source.
Certificate Revocation List (CRL) — Intel's CRL is needed to verify the TDX attestation quote. It is served by this server, which pre-fetches it from Intel periodically. Its authenticity is verified cryptographically as part of the quote verification below.

Verify Quote

This verifies the authenticity of the remote server by fetching and checking its TDX attestation quote. The verification confirms two things: that the server is running inside a genuine Intel TDX trusted execution environment, and that the code running inside it matches the expected measurement (RTMR2).

Expected RTMR2: —

The RTMR2 value is a measurement of the code running inside the TDX enclave. This expected value is embedded in this client at build time. If you do not trust this server, you should obtain the expected RTMR2 from a trusted channel and compare it with the value shown above.

Please verify the tool and its parameters before proceeding. The RTE guarantees isolation of the analysis environment, but does not control the behaviour of the selected tool — you must independently trust the tool you choose to run.
The RTE provides a trusted execution environment that isolates the analysis from external observation. However, the RTE does not audit or endorse any specific tool. The tool operates on your data within the enclave, so you should only select tools you trust to handle your assets appropriately.
Your file will be encrypted locally on your device before upload. It can only be decrypted inside the verified trusted enclave — no one else can access it, not even us.
Encryption is performed in your browser using a public key extracted from the server's attested TDX quote. The corresponding private key exists only inside the enclave and is never exposed. This ensures that your data remains confidential in transit and at rest, and can only be processed within the isolated environment whose integrity was verified in the previous step.
